## Git conventions
- **Commit format:** Conventional Commits. **Scope:** the module/package touched (e.g.
  `changelog`, `provenance`); release commits use `chore(release): vX.Y.Z`.
- **Main branch:** master  •  **Feature branches:** `<branch>-<tag>` conventions apply to dist-tags.
- Version bumps are driven by commit type on push: `feat` → minor, `fix`/others → patch,
  `BREAKING CHANGE`/`!` → major.

## Special rules
- **🚨 Pushing to `master` (or `main`) auto-publishes to npm.** `.github/workflows/release.yml`
  runs on every push to those branches: it bumps the version from conventional commits, tags, and
  runs `npm publish --provenance`. **Never push without explicit user intent** — a push is a
  release. Prefer local commits and let the user push.
- Provenance in CI requires `id-token: write` permission and a `repository` field in each
  published `package.json`; `lockstep publish --provenance` validates this fail-fast before
  publishing anything.
- All git operations in `lockstep.ts` run against the configured repository root
  (`this.config.root`, default `process.cwd()`), not the process working directory.
